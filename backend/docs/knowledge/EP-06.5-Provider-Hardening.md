## Changes by Recommendation

### REC-01 â€” HealthCheckInterface Inheritance

**File**: `app/providers/interface.py`

`AIProvider` now inherits from `HealthCheckInterface` instead of defining health-check methods independently. This eliminates duplicate abstract signatures and makes the architectural contract explicit: any component that depends on health checking can accept `HealthCheckInterface` without depending on the full `AIProvider`.

```python
class AIProvider(HealthCheckInterface):   # was: class AIProvider(ABC)
    ...
```

All 7 adapters satisfy both interfaces automatically.

### REC-02 â€” SSRF Validation

**File**: `app/providers/config.py`

`_check_ssrf(url, *, allow_http)` validates provider `base_url` and `azure_endpoint` at model construction. No network calls â€” pure string and `ipaddress` analysis.

Rules for cloud providers (`allow_http=False`):
- Scheme must be `https`
- Hostname must not be `localhost` / `localhost.localdomain`
- IP addresses must not be loopback, private, link-local, or reserved
- Cloud metadata hosts always blocked: `169.254.169.254`, `fd00:ec2::254`, `metadata.google.internal`, `metadata.internal`

Rules for self-hosted providers (`allow_http=True`, e.g. Ollama):
- `http` permitted
- Private / loopback IPs permitted (Ollama on LAN is a valid enterprise setup)
- Cloud metadata hosts still always blocked

The opt-in is `_allow_http_base_url: ClassVar[bool] = False` on `ProviderConfig`, overridden to `True` on `OllamaConfig`. No subclass can accidentally inherit the permissive mode unless it explicitly sets the flag.

`AzureOpenAIConfig` has a dedicated `@field_validator("azure_endpoint")` calling `_check_ssrf(v, allow_http=False)`.

### REC-03 â€” Post-Construction Registry Verification

**File**: `app/providers/factory.py`, `app/providers/errors.py`

`ProviderFactory.create()` now verifies that the instantiated adapter's `provider_type` property matches the registry key it was looked up under:

```python
if instance.provider_type != provider_type:
    raise ProviderConfigurationError(
        f"Registry misconfiguration: {cls.__name__} is registered under "
        f"{provider_type!r} but its provider_type property reports "
        f"{instance.provider_type!r}. ..."
    )
```

This catches copy-paste errors (e.g. `AnthropicProvider` accidentally registered under `ProviderType.OPENAI`) that would otherwise cause silent wrong-provider execution.

`ProviderConfigurationError(ProviderError)` is a new non-retryable error type.

### REC-04 â€” `provider_type` Field Validation

**File**: `app/providers/config.py`

`ProviderConfig.provider_type` is validated against the `ProviderType` enum at construction time via a `@field_validator`. Invalid strings are rejected immediately with an error message listing all valid values. Subclasses with hardcoded defaults (e.g. `OpenAIConfig.provider_type = "openai"`) pass through the validator transparently.

Lazy import inside the validator body breaks the potential circular import between `app.providers.config` and `app.models.provider_connection`.

### REC-05 â€” `get_usage()` on All Adapters

**File**: `app/providers/interface.py` + all 7 adapters

`get_usage(start_date, end_date)` is now an abstract method on `AIProvider`. All 7 adapters implement it:

- OpenAI, Anthropic, Grok, Google, Azure OpenAI, OpenRouter: raise `NotImplementedError("... implemented in EP-08")`
- Ollama: raises `NotImplementedError("Ollama is self-hosted and does not expose a usage API")`

`check_capability(capability)` and `is_healthy` were also missing from some adapters; all 7 now implement both. `is_healthy` returns `False` conservatively (not connected = not healthy). `check_capability` raises `NotImplementedError("... implemented in EP-07")` since it requires real provider queries.

### REC-06 â€” Message Content Type Hierarchy

**File**: `app/providers/models.py`

`ProviderRequest.messages` was `list[dict[str, str]]`. It is now `list[Message]`.

`Message` accepts either a plain string or a list of typed content blocks:

```python
class Message(BaseModel):
    role: MessageRole
    content: str | list[MessageContent]
    name: str | None = None
    tool_call_id: str | None = None
```

`MessageContent` is a Pydantic v2 discriminated union keyed on `Literal["type"]`:

| Class | `type` value | Purpose |
|---|---|---|
| `TextContent` | `"text"` | Plain text |
| `ImageUrlContent` | `"image_url"` | Image by URL |
| `ImageBase64Content` | `"image_base64"` | Inline base64 image |
| `AudioContent` | `"audio"` | Inline base64 audio |
| `ToolCallContent` | `"tool_call"` | Model emits tool calls |
| `ToolResultContent` | `"tool_result"` | Caller returns tool result |

Existing code passing `[{"role": "user", "content": "hello"}]` continues to work because Pydantic v2 auto-coerces dicts to `Message`.

### REC-07 â€” `SecretStoreType` Enum + Dead Code Removal

**File**: `app/providers/config.py`

`SecretReference.secret_store` was typed as `str`. It is now `SecretStoreType`:

```python
class SecretStoreType(enum.StrEnum):
    ENV = "env"
    VAULT = "vault"
    AWS_SECRETS_MANAGER = "aws_secrets_manager"
```

`OllamaConfig` had a dead `@validator` that always passed without side effects; it has been removed. The `_allow_http_base_url: ClassVar[bool] = True` ClassVar is the correct mechanism for the SSRF opt-in.
